FAQ

Cannot log in to the admin panel

Possible causes and solutions:

  • Service not started: Verify the EasyMail process is running and there are no errors in the logs
  • Port not open: Check if the firewall allows traffic on port 8000
  • Default admin password: Find the admin password field in the startup logs to get the initial password
  • If you forgot the password, you can reset the admin password directly through the database
Client IMAP connection fails

Troubleshoot in the following order:

  • Confirm the IMAP service is running (check the dashboard service status or EasyMail logs)
  • Confirm the mailbox account has been created and its status is "enabled"
  • Confirm you are using the full email address as the username (e.g. user@example.com, not just user)
  • Confirm the firewall allows port 143 (and port 993 if using SSL)
  • Check EasyMail logs for IMAP connection failure error messages
  • Test locally on the server using telnet: telnet 127.0.0.1 143 to see if a connection can be established
Note EasyMail's IMAP supports AUTHENTICATE PLAIN and LOGIN authentication methods. If the client is configured for secure authentication (e.g. CRAM-MD5), switch to plain password authentication.
Email sending fails (SMTP rejected)

Possible causes:

  • Postfix is not properly configured or not running
  • SMTP authentication fails: Verify the username (full email address) and password are correct
  • The sender's domain has not been added to EasyMail
  • Postfix's smtpd_recipient_restrictions is rejecting the connection
  • Email was rejected by Milter filtering (check the delivery logs for filtering results)
  • Port blocked: Port 25 is often blocked by ISPs; it is recommended to use port 587

Troubleshooting: Check Postfix logs at /var/log/mail.log and the Milter decision in EasyMail logs.

Cannot receive incoming external emails

Troubleshooting chain:

  • Check if the DNS MX record points correctly to the Postfix server
  • Confirm from Postfix logs that external senders are successfully connecting to the server
  • Check EasyMail delivery logs to see if the email was intercepted by filtering rules (spam / quarantine / reject)
  • Check the Spam and Quarantine folders for any misclassified emails
  • Check if there is sufficient disk storage space
  • Confirm the mailbox capacity has not been exceeded
Foxmail / Outlook client disconnects immediately after connecting

This is usually an IMAP protocol compatibility issue. Please check:

  • Whether EasyMail is up to date (compatibility issues may have been fixed in later versions)
  • Check EasyMail logs for the specific error message regarding the IMAP disconnection
  • IMAP response format must strictly follow RFC 3501 specifications; some clients are sensitive to formatting
  • Do not use BODY.PEEK as a data item name in FETCH responses; use BODY instead
  • Verify that the RECENT response value is not 0 (if emails already exist)
Milter filtering causes false positives on legitimate emails

Solutions:

  • Adjust the priority or threshold of rules on the "Filter Rules" page
  • Move misclassified emails from the spam folder back to the inbox; the system will learn from this
  • Add the sender to the whitelist (via custom rules)
  • Adjust the threshold of the ML classification model
  • Collect false positive samples to retrain the classification model
  • Temporarily disable the rule causing false positives to identify the problematic rule
How to back up and restore email data

EasyMail's email data consists of three parts:

  • MySQL Database: domains, accounts, filter rules, classification models, and other management data
    Backup method: mysqldump -u root -p easymail > easymail_backup.sql
  • Disk Storage: raw .eml files (located under the configured storage_root directory)
    Backup method: directly copy the directory, e.g. rsync -av /var/easymail/data/ /backup/
  • Redis Data: cache and statistics (non-essential, can be automatically rebuilt if lost)

Restore procedure: First restore the MySQL data, then restore the disk files, and finally start EasyMail. Redis data will be rebuilt automatically.

How to upgrade EasyMail
# Pull the latest code
git pull

# Rebuild
go build -o easymail ./cmd/easymail

# Back up the current binary
cp /usr/local/bin/easymail /usr/local/bin/easymail.bak

# Replace the binary
cp easymail /usr/local/bin/easymail

# Restart the service
systemctl restart easymail
Note It is recommended to back up MySQL and the storage directory before upgrading. The database.automigrate: true config setting will automatically apply database schema changes on startup.
How to view system logs

EasyMail outputs logs to stdout by default. If using systemd:

journalctl -u easymail -f

Log level is configured in easymail.yaml:

  • debug: detailed debugging information (use when troubleshooting)
  • info: general runtime information (default)
  • warn: warnings and errors only
  • error: errors only

Log format example:

time="2026-07-08T10:30:00+08:00" level=info msg="imap: connection from 192.168.1.100:54321" module=imap
time="2026-07-08T10:30:01+08:00" level=info msg="lmtp: delivered mail to user@example.com" module=lmtp
time="2026-07-08T10:30:02+08:00" level=debug msg="[IMAP-DEBUG] C: C1 LOGIN user@example.com ***" module=imap
How to configure HTTPS

EasyMail's admin panel and Webmail use plain HTTP by default. It is recommended to configure HTTPS in production:

Option 1 (Recommended): Use a reverse proxy (Nginx / Caddy)

# Nginx configuration example
server {
    listen 443 ssl;
    server_name admin.example.com;

    ssl_certificate /etc/ssl/certs/example.com.pem;
    ssl_certificate_key /etc/ssl/private/example.com.key;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Option 2: Support TLS at the application level (requires implementing certificate loading configuration yourself).

Which databases does EasyMail support?

Via the GORM ORM, it supports:

  • MySQL 8.0+ (default, recommended)
  • PostgreSQL (change the driver type in the configuration file)
  • SQLite3 (testing only, not recommended for production)

Modify in the configuration file:

database:
  driver: mysql       # options: mysql / postgres / sqlite
  dsn: "..."
