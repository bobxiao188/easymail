Admin Panel Guide

The admin panel is the core operational interface of EasyMail, providing domain management, mailbox management, filter rules, classification models, delivery logs, and more. It listens on port 8000 by default.

Dashboard

The home page after login is the dashboard, which displays key operational information:

  • Service Status: The running status and last heartbeat time of each component (Dovecot, LMTP, Milter, Admin, Webmail, IMAP, Classifier)
  • Today's Mail Statistics: Total emails received, filtered count, rejected count, spam count
  • Top Senders: Top 10 senders by volume today
  • Recent Activity: Recent delivery records and system events

Domain Management

Add Domain

Go to the Domain Management page and click "Add Domain". You need to fill in:

  • Domain: The mail domain, e.g. example.com
  • Description: Remarks (optional)

After adding, the system will create the domain's management record in MySQL and prepare the storage directory structure.

Domain Status

StatusDescription
EnabledThe domain can receive mail normally
DisabledThe domain is suspended from receiving mail; Postfix queries will return "not found"
DeletedMarked for deletion, resources are not immediately released

DKIM Configuration

In the domain details, you can generate and manage DKIM key pairs. DKIM is used to sign outgoing emails, and recipients can verify the email's authenticity by querying the public key via DNS.

  • Click "Generate DKIM Key Pair"
  • The system generates an RSA public/private key pair, and the public key is displayed in DNS TXT record format
  • Add the TXT record to the domain's DNS: default._domainkey.example.com TXT "v=DKIM1; k=rsa; p=public-key-content"
  • Enable DKIM signing on outbound emails in Postfix or EasyMail

Domain Operations

  • Purge: Delete all mail data for this domain (disk + SQLite index) — use with caution
  • Enable/Disable: Toggle domain status; when disabled, all mailboxes under this domain cannot send or receive mail

Mailbox Management

Add Mailbox

Go to the Mailbox Management page and click "Add Mailbox":

  • Email Address: Full email address, e.g. user@example.com
  • Password: IMAP login password
  • Domain: Select an already added domain
  • Mailbox Capacity: Disk space limit (optional, 0 means unlimited)

Mailbox Status

StatusDescription
EnabledThe mailbox can send and receive mail normally
DisabledThe mailbox is suspended; IMAP login and mail delivery are both blocked

Mailbox Operations

  • Change Password: Reset the mailbox login password
  • Purge Mailbox: Delete all mail data under this mailbox
  • Enable/Disable: Control the mailbox availability status

Filter Rule Management

Filter Pipeline

EasyMail's anti-spam filtering integrates with Postfix via the Milter protocol, processing each inbound email in 6 stages:

  1. Connect Stage: Records the connecting IP and performs IP reputation checks
  2. Helo Stage: Validates the client HELO/EHLO hostname and checks reverse DNS
  3. MailFrom Stage: Validates the sender's domain and checks SPF records
  4. RcptTo Stage: Validates whether the recipient exists and checks against the contact list
  5. Headers Stage: Analyzes email headers (Received, DKIM-Signature, etc.), removes inbound EasyMail policy headers
  6. Body Stage: MIME parsing, text/HTML extraction, URL feature analysis, attachment risk assessment, ML classification inference, ClamAV virus scanning, rule engine final verdict

Filter Results

ResultActionDescription
acceptDeliver normallyEmail goes to inbox
spamMark as spamEmail delivered to spam folder
quarantineQuarantineEmail held in quarantine, pending manual review by administrator
rejectSMTP rejectReturns a 5xx error code directly, email is not stored

Create a Filter Rule

On the Filter Rules page, click "Add Rule":

  • Name: A descriptive name for the rule
  • Condition: Select a feature extractor and comparison condition. For example:
    • Connecting IP is in a known malicious IP list
    • HELO name does not match reverse DNS
    • Sender domain has no MX record
    • Email body contains specific keywords
    • URL links point to known malicious sites
    • Attachment type is a high-risk format
  • Action: The handling method when matched (Accept / Mark as Spam / Quarantine / Reject)
  • Priority: Lower numbers mean higher priority; rules are evaluated in priority order
  • Enabled Status: Allows temporarily disabling a rule

ML Classification Model

The Classification Model page manages the machine learning classification engine:

  • Model Types: FastText (word-vector-based text classification), DistilBERT (Transformer-based deep classifier), XGBoost (pending implementation)
  • Model Management: Import, export, train, and delete models
  • Sample Management: Manage global training data on the Training Samples page for model training
  • Training: Train the classification model using sample data; the model takes effect automatically after training
  • Prediction: Predict against specified email content to verify model performance
Tip ML classification is optional. If no model is configured, the system will only use the rule engine for filtering.

Delivery Logs

The Delivery Logs page records the processing details of each email:

  • Sender, recipient
  • Email subject
  • Filtering time
  • Filter result (accept / spam / quarantine / reject)
  • Matched rules
  • Feature extraction snapshot
  • Delivery status

Logs can be filtered by time range, sender, recipient, and filter result for easy troubleshooting.

Administrator Account

On the Profile page, you can manage the current admin account:

  • Change name and email
  • Change password
  • Switch interface language (supports Chinese/English)
  • Switch interface theme

Postfix Management Page

If Postfix Agent is deployed, the admin panel also provides the following Postfix management features:

  • Service Status: View Postfix running status and configuration version
  • Configuration Management: Remotely manage Postfix configuration parameters, supports configuration deployment and rollback
  • Agent Management: Manage Postfix Agent servers
  • Queue Management: View the Postfix mail queue

For more details, please refer to Postfix Integration.
