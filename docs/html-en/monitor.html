Service Monitoring

EasyMail uses a heartbeat mechanism to monitor the running status of each service component. Each enabled service periodically writes a heartbeat timestamp to Redis, and the admin panel reads the heartbeat records from Redis to determine the health status of each service.

Heartbeat Mechanism

The heartbeat mechanism workflow:

Startup
  │
  ├── Create global ServiceManager
  ├── Register each service (Dovecot, LMTP, Milter, Admin, Webmail, IMAP, Classifier)
  │     └── Each service has its own heartbeat interval (defaults to global config)
  └── Start heartbeat manager
        │
        Every 10 seconds (configurable):
        ├── Iterate through all registered services
        ├── Write current timestamp to Redis
        │     └── key:  easymail:service:{service_name}:heartbeat
        │         value: ISO8601 timestamp
        └── Advance to next cycle

Heartbeat configuration in easymail.yaml:

# Global heartbeat config
heartbeat_second: 10  # Heartbeat interval (seconds), set to 0 or when Redis is unavailable to auto-disable

# Individual service config (automatically inherits global heartbeat settings)
imap:
  enable: true

milter:
  enable: true
Note Heartbeat is automatically enabled (when heartbeat_second > 0 and Redis is available), no need to configure per service.

Admin Dashboard

After logging into the admin panel, the dashboard shows the running status of each service:

StatusCriteriaIndicator
runningLast heartbeat < 30 secondsGreen
warningLast heartbeat 30-120 secondsYellow
stoppedLast heartbeat > 120 seconds or no heartbeat recordRed

The dashboard also displays:

  • Today's Mail Statistics: total received, filtered, rejected, spam count
  • Top Senders: today's TOP10 senders by volume
  • Service Status: overview of all services

CLI Heartbeat Tool

EasyMail includes the heartbeat-cli command-line tool to quickly view the heartbeat status of all services:

cd cmd/heartbeat-cli
go run main.go -config /path/to/easymail.yaml

Sample output:

----------------------------------------------------------------------

API Query

The admin panel's GET /api/v1/dashboard/services endpoint returns service status JSON, suitable for integration with external monitoring systems:

GET /api/v1/dashboard/services
Authorization: Bearer {token}

Response:
{
  "code": 0,
  "data": {
    "dovecot": { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "lmtp":    { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "milter":  { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "admin":   { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "webmail": { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "imap":    { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" }
  }
}

Direct Redis Query

If you need to manually or via script check heartbeat status:

# List all registered services
redis-cli keys "easymail.service:*"

# View a specific service's heartbeat
redis-cli get "easymail.service:imap:heartbeat"

# Sample output
# "2026-07-08T10:30:45Z"

Mail Statistics

The admin panel API also provides mail traffic statistics:

  • GET /api/v1/dashboard/stats: Returns today's mail statistics (total, filtered, rejected, spam counts)
  • GET /api/v1/dashboard/top-senders: Returns today's TOP10 senders by volume

These statistics are computed in real-time via Redis counters, without database queries, for fast response.
