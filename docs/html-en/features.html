Built-in Feature List

EasyMail's filtering engine provides the following built-in features, which can be directly referenced in the condition expressions of filter rules. Features are organized by function category, with each feature annotated with its data type, processing stage, and a brief description.

Note The "Stage" in the table below corresponds to the SMTP processing pipeline: 0=Connection Setup, 1=HELO, 2=Sender, 3=Recipient, 4=Headers, 5=Body. Features within the same category may be distributed across multiple stages.

1. Connection Reputation

Collected during the SMTP connection establishment phase, evaluating the network identity and reputation of the client.

Using Features in Filter Rules

On the "Filter Rules" page in the admin panel, when creating or editing a rule, the "Select Feature" dropdown in the condition editor includes all of the features listed above.

Rule condition examples:

# Reject emails that fail SPF verification
condition: spf_fail == true
action: reject

# Mark as spam for emails from non-local contacts containing short links
condition: rcpt_contact_sender_hit == false AND body_url_has_short == true
action: spam

# Directly reject high-frequency connections with no PTR record
condition: ip_ptr_ok == false AND ip_connect_count_5m > 50
action: reject

# Quarantine when DMARC fails and DKIM does not pass
condition: dmarc_fail == true AND dkim_fail == true
action: quarantine

Rules are evaluated in priority order; execution stops at the first match (first matching rule takes effect).
