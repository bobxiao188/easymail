Quick Start

This guide walks you through the complete process from deployment to successfully sending and receiving emails. If you have already completed the installation and deployment, you can start directly from Step 1.

Workflow Diagram

Admin Panel (8000)                    Postfix Server
    │                                 │
    ├─ 1. Add Mail Domain ────────────┤
    ├─ 2. Create Mailbox ─────────────┤
    ├─ 3. Configure Postfix main.cf ──┤
    └─ 4. Verify Mail Delivery ───────┴─→ 5. Configure Mail Client
1

Add a Mail Domain

Log in to the admin panel at http://ServerIP:8000 and navigate to the Domain Management page.

Click "Add Domain" and fill in the following information:

  • Domain: Your mail domain, e.g. example.com
  • Description: Optional notes

After adding successfully, a new entry will appear in the domain list. The domain status will be "Enabled". You need to configure MX records for this domain to point mail to your Postfix server:

example.com.  IN  MX  10  mail.example.com.
mail.example.com.  IN  A   Your Server Public IP
Tip If this is just for internal testing, you can skip DNS configuration and simply allow internal sending IPs in Postfix's mynetworks.
2

Create a Mailbox

Go to the Mailbox Management page (or click the "Mailboxes" tab in the domain details).

Click "Add Mailbox" and fill in:

  • Email Address: e.g. user@example.com
  • Password: Set the login password
  • Domain: Select the domain added in the previous step
  • Mailbox Capacity: Limit the disk space for the mailbox

After creation, the mailbox can be used for IMAP login and receiving mail.

Note After creating a mailbox, the system will automatically create the user's folder structure (Inbox, Sent, Spam, Trash, etc.) in the storage directory.
3

Configure Postfix

Edit /etc/postfix/main.cf on the Postfix server:

# Mail domain list (synced from EasyMail admin panel)
virtual_mailbox_domains = tcp:192.168.1.1:10025
virtual_mailbox_maps = tcp:192.168.1.1:10025

# LMTP delivery to EasyMail
virtual_transport = lmtp:inet:192.168.1.1:10027

# Milter anti-spam filtering
smtpd_milters = inet:192.168.1.1:10026
milter_default_action = accept

# SMTP authentication (via EasyMail's Dovecot protocol proxy)
smtpd_sasl_type = dovecot
smtpd_sasl_path = inet:192.168.1.1:10025
smtpd_sasl_auth_enable = yes
smtpd_recipient_restrictions =
    permit_mynetworks,
    permit_sasl_authenticated,
    reject_unauth_destination
smtpd_sasl_security_options = noanonymous
local_recipient_maps =

Replace 192.168.1.1 with the actual IP address of your EasyMail server.

After modification, restart Postfix:

systemctl restart postfix
Note EasyMail's Dovecot authentication proxy listens on TCP port 10025 and provides both domain lookup and authentication services. Postfix queries domain existence via the tcp: protocol and performs SMTP authentication via dovecot SASL.
4

Verify Mail Delivery

Verification steps:

4.1 Confirm Postfix recognizes the mail domain

postmap -q example.com tcp:192.168.1.1:10025

This should output OK or information about the mail domain.

4.2 Send a test email

Send a test email from an external mailbox (e.g., Gmail, QQ Mail) to user@example.com. Or use the command line:

echo "Test email body" | mail -s "Test Subject" user@example.com

Check the EasyMail logs; you should see output similar to:

time="..." level=info msg="milter: filter result=accept" module=milter
time="..." level=info msg="lmtp: delivered mail to user@example.com" module=lmtp

4.3 Check if the email arrived

View the delivery records in the admin panel's Delivery Logs. The logs display the sender, recipient, filter result, and delivery status for each email.

4.4 Check with an IMAP client

Use any IMAP client to connect to the EasyMail server (port 143) and check the inbox. Configuration parameters:

  • IMAP Server: Your server IP
  • IMAP Port: 143 (plain) or 993 (SSL)
  • SMTP Server: Your Postfix server IP
  • SMTP Port: 587 (STARTTLS) or 25
  • Username: The full email address, e.g. user@example.com
  • Password: The password set when creating the mailbox

For detailed client configuration, refer to the Client Configuration Guide.

5

Configure Anti-Spam Filtering

After the mail system is working normally, it is recommended to configure anti-spam filtering policies. Go to the Filter Rules page in the admin panel:

  • Review the default filter rules
  • Adjust rule priorities as needed
  • Configure classification models (optional, requires prior training)
  • Enable ClamAV virus scanning (optional, requires ClamAV installation)

For detailed configuration, refer to the filtering section in the Admin Panel Guide.
