Obtaining EasyMail

Building from Source

git clone https://github.com/your-org/easymail.git
cd easymail
go build -o easymail ./cmd/easymail

The build produces a single binary easymail (or easymail.exe) containing all built-in services and protocol adapters.

Configuration File

EasyMail uses a YAML configuration file. Create easymail.yaml:

# Basic Configuration
server_name: mail.example.com
storage_root: /var/easymail/data

# Database
database:
  driver: mysql
  dsn: "root:password@tcp(127.0.0.1:3306)/easymail?charset=utf8mb4&parseTime=True&loc=Local"
  automigrate: true

# Redis (optional but recommended)
redis:
  addr: "127.0.0.1:6379"
  password: ""
  db: 0

# Service Ports
ports:
  admin: 8000      # Admin Panel
  webmail: 8080    # Webmail
  imap: 143        # IMAP
  imaps: 993       # IMAP over TLS
  lmtp: 10027      # LMTP Delivery
  milter: 10026    # Milter Filtering
  dovecot: 10025   # Dovecot Auth Proxy

# Admin panel static file path
admin_static: ./frontend/admin/dist

# Logging
log_level: info    # debug / info / warn / error

# Anti-Spam Module
filter:
  # ... see Anti-Spam Filtering section

heartbeat_second: 10  # Heartbeat interval
Tip If database.automigrate: true is enabled, EasyMail will automatically create all database tables on first startup.

Starting the Service

./easymail -config easymail.yaml

After starting, EasyMail initializes the database connection, creates table structures, loads the cache, and then starts all configured services. Example log output:

Verifying Service Status

Once started, you can check the status of all services through the admin panel:

Open your browser and visit http://SERVER_IP:8000. Log in for the first time using the default admin account:

  • Default account: admin@easymail.local
  • Default password: look for the admin password field in the startup logs
Security Notice Change the admin password immediately after the first login. For production environments, configure HTTPS (reverse proxy or direct TLS certificate configuration).

After logging in, the dashboard displays the running status of all services:
