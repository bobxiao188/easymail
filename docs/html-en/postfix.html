Postfix Integration

EasyMail does not handle SMTP inbound; it relies on Postfix as the edge MTA. Postfix forwards mail to EasyMail for anti-spam filtering via the Milter protocol, and filtered mail is delivered to EasyMail storage via LMTP.

Integration Architecture

Sender ──SMTP──→ Postfix (25/587)
                      │
                      ├── Milter (10026) ──→ EasyMail Filtering Decision
                      │                        └── accept / reject / spam / quarantine
                      │
                      └── LMTP (10027) ──→ EasyMail Delivery
                                             └── Write SQLite Index + Disk Storage

Installing Postfix

apt install postfix
# Select "Internet Site" during installation, enter your domain

Configuring main.cf

Edit /etc/postfix/main.cf and append the following configuration:

# Mail domain lookup (via EasyMail Dovecot protocol proxy)
virtual_mailbox_domains = tcp:192.168.1.1:10025
virtual_mailbox_maps = tcp:192.168.1.1:10025

# LMTP delivery (hand mail over to EasyMail storage)
virtual_transport = lmtp:inet:192.168.1.1:10027

# Milter anti-spam filtering
smtpd_milters = inet:192.168.1.1:10026
milter_default_action = accept

# SMTP authentication (allow clients to send mail)
smtpd_sasl_type = dovecot
smtpd_sasl_path = inet:192.168.1.1:10025
smtpd_sasl_auth_enable = yes
smtpd_sasl_security_options = noanonymous

# Recipient restrictions
smtpd_recipient_restrictions =
    permit_mynetworks,
    permit_sasl_authenticated,
    reject_unauth_destination

# Do not rely on local user database
local_recipient_maps =
Important Replace 192.168.1.1 with the actual IP address of your EasyMail server. The EasyMail Dovecot protocol proxy provides three services simultaneously: mail domain lookup, SASL authentication, and LMTP delivery.

Configuration Details

ParameterValuePurpose
virtual_mailbox_domainstcp:IP:10025Postfix queries EasyMail via TCP to verify whether a domain is valid
virtual_mailbox_mapstcp:IP:10025Postfix checks whether a mailbox address exists in EasyMail
virtual_transportlmtp:inet:IP:10027Delivers mail to EasyMail via LMTP
smtpd_miltersinet:IP:10026Milter filter chain; EasyMail analyzes the mail and returns a decision
smtpd_sasl_*—SMTP authentication; clients must log in with their mailbox account to send mail
smtpd_recipient_restrictions—Recipient restriction rules, evaluated in order

master.cf

The default master.cf works out of the box without modification.

Verifying the Integration

1. Verify Domain Lookup

postmap -q example.com tcp:192.168.1.1:10025

Should return OK (if the domain has been added in the EasyMail admin panel).

2. Verify Mailbox Lookup

postmap -q user@example.com tcp:192.168.1.1:10025

Should return OK (if the mailbox has been created).

3. Send a Test Email

echo "Hello from test" | sendmail user@example.com

Check EasyMail logs to confirm the mail was delivered.

4. Check Postfix Logs

tail -f /var/log/mail.log

Common Postfix Configurations

Allow Internal Network Sending

mynetworks = 127.0.0.0/8 192.168.0.0/16 10.0.0.0/8

Limit Single Email Size

message_size_limit = 25600000  # 25MB

Concurrent Connection Limits

smtpd_client_connection_count_limit = 10
smtpd_client_message_rate_limit = 30

TLS Configuration (587 STARTTLS)

smtpd_tls_cert_file = /etc/ssl/certs/postfix.pem
smtpd_tls_key_file = /etc/ssl/private/postfix.key
smtpd_tls_security_level = may
smtpd_tls_loglevel = 1

Postfix Agent Remote Management

EasyMail ships with a postfix-agent tool that runs on the Postfix host and communicates with the EasyMail admin panel via HTTP, enabling remote management of Postfix instances.

How It Works

postfix-agent starts an HTTP service on the Postfix host and actively heartbeats to the EasyMail admin panel. The agent authenticates using --agent-token; only agents with a valid token can be recognized and controlled by the admin panel.

Postfix Host                          EasyMail Server
┌──────────────────┐     HTTP/Heartbeat ┌────────────────────┐
│  postfix-agent   │ ────────────────→ │  Admin Panel Web   │
│  --agent-token   │ ←──────────────── │  + API Service     │
│                  │   Config/Command  │                    │
│  Operate postfix │                   │  Admin Web Ops     │
│  (postqueue,     │                   │                    │
│   postsuper,     │                   │                    │
│   postfix reload)│                   │                    │
└──────────────────┘                   └────────────────────┘

Deployment

Download the postfix-agent binary distributed by EasyMail onto your Postfix host, then run:

./postfix-agent --agent-token=your-agent-token-here

Parameters:

FlagDescription
--agent-tokenAgent identity token, generated in the admin panel for authentication
--easymail-addrEasyMail server address (default http://localhost:8080)
--listenLocal listen address (default :8081, for health checks)

It is recommended to manage the agent process with systemd or supervisord to ensure it starts on boot. After startup, you should see output similar to:

[INFO] Registering with EasyMail at http://192.168.1.1:8080
[INFO] Agent registered successfully, id: agent-xxxx

Admin Panel — Adding a Postfix Agent

In the EasyMail admin panel:

  1. Go to System Settings → Postfix Agents
  2. Click Add Agent
  3. Enter a name for the agent (e.g. "Primary Postfix Node")
  4. The system generates an agent_token (e.g. ema_xxxxxxxxxxxx)
  5. Copy this token to the postfix-agent --agent-token=... command on your Postfix host
  6. After saving, the agent appears in the list with status Waiting for Connection

Once the agent starts and registers successfully, the status changes to Online, displaying the last heartbeat time, IP address, and other info.

Admin Panel — Applying EasyMail Host Address to Postfix Config

After the agent is registered, go to Postfix Configuration Management and select a registered agent as the target deployment host:

  1. Go to System Settings → Postfix Configuration
  2. Edit or create a Postfix configuration (main.cf / master.cf snippet)
  3. In the Target dropdown, select a registered agent (displayed as "Agent Name - IP Address")
  4. The system automatically injects the EasyMail Host Address into the configuration template to generate the final main.cf

Use the {{.EasyMailHost}} placeholder in your templates to reference the EasyMail server address:

# Placeholders in the template are replaced with the actual EasyMail address during push
virtual_mailbox_domains = tcp:{{.EasyMailHost}}:10025
virtual_mailbox_maps = tcp:{{.EasyMailHost}}:10025
virtual_transport = lmtp:inet:{{.EasyMailHost}}:10027
smtpd_milters = inet:{{.EasyMailHost}}:10026

Push and Deploy Configuration

After editing the configuration:

  1. Click Deploy on the Postfix Configuration page
  2. Select the target agent(s) (check the Postfix hosts to deploy to)
  3. Review the configuration diff — the system shows what will be added, modified, or removed
  4. Click Confirm Push

Deployment flow:

Admin Panel                   Agent                       Postfix
   │                           │                           │
   ├── Push config ──────────→ │                           │
   │                           ├── Write to /etc/postfix/  │
   │                           ├── Validate config         │
   │                           │   postfix check           │
   │                           ├── Reload service          │
   │                           │   postfix reload          │
   │                           ├── Return result ────────→ │
   │                           │                           │
   │  ── Show deploy result ── │                           │
   │  Success/Failure + Logs   │                           │

Every push is recorded in the deployment history, including timestamp, operator, configuration summary, and execution result.

Queue Management

Through the agent, you can remotely view and manage the Postfix mail queue from the admin panel:

Viewing Queue Status

  1. Go to System Settings → Postfix Agents
  2. Click Queue Management on the target agent
  3. The system displays a queue overview:
Queue TypeDescription
activeMail currently being delivered
deferredMail that failed delivery, waiting for retry
bounceBounced mail queue
holdMail placed on hold
incomingMail currently being received

Queue Operations

The following remote operations are supported:

  • Flush Queue — Immediately retry all mail in the deferred queue (equivalent to postqueue -f)
  • Delete Mail — Delete a specific mail by queue ID (equivalent to postsuper -d <queue_id>)
  • Hold / Release — Move mail to the hold queue or release it from hold (equivalent to postsuper -h/-H <queue_id>)
  • Purge Queue — Delete all mail in a specific queue (equivalent to postsuper -d ALL)

All queue operations are logged in the audit trail.

Real-time Logs

The queue management page also displays real-time logs from the agent, making it easier to diagnose delivery issues:

2026/07/12 10:23:45 [mail.log] postfix/qmgr[12345]: AABBCCDD: from=<user@example.com>,
    size=12345, nrcpt=1 (queue active)
2026/07/12 10:23:46 [mail.log] postfix/smtp[12346]: AABBCCDD: to=<dest@other.com>,
    relay=mail.other.com[1.2.3.4]:25, delay=0.5, status=sent (250 2.0.0 Ok)
