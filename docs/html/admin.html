管理后台功能指南

管理后台是 EasyMail 的核心操作界面，提供域名管理、邮箱管理、过滤规则、分类模型、投递日志等功能。默认监听 8000 端口。

控制面板

登录后首页为控制面板，展示系统运行的关键信息：

  • 服务状态：各组件（Dovecot、LMTP、Milter、Admin、Webmail、IMAP、Classifier）的运行状态和最后心跳时间
  • 今日邮件统计：收到的邮件总数、已过滤数、拒绝数、垃圾邮件数
  • 热门发件人：今日发件量最多的发件人 TOP10
  • 最近活动：最近的投递记录和系统事件

域名管理

添加域名

进入【域名管理】页面，点击"添加域名"。需要填写：

  • 域名：邮件域名，例如 example.com
  • 描述：备注信息（可选）

添加后，系统会在 MySQL 中创建该域名的管理记录，并准备存储目录结构。

域名状态

状态说明
启用域名可正常接收邮件
禁用域名暂停邮件接收，Postfix 查询时将返回不存在
删除标记删除，不立即释放资源

DKIM 配置

在域名详情中，可以生成和管理 DKIM 密钥对。DKIM 用于对发出的邮件进行签名，接收方可以通过 DNS 查询公钥验证邮件真实性。

  • 点击"生成 DKIM 密钥对"
  • 系统生成 RSA 公私钥，公钥以 DNS TXT 记录格式展示
  • 将 TXT 记录添加到域名的 DNS 中：default._domainkey.example.com TXT "v=DKIM1; k=rsa; p=公钥内容"
  • 在 Postfix 或 EasyMail 的出站邮件中启用 DKIM 签名

域名操作

  • 清除：删除该域名下的所有邮件数据（磁盘 + SQLite 索引），谨慎操作
  • 启用/禁用：切换域名状态，禁用时该域名下的所有邮箱不可收发邮件

邮箱管理

添加邮箱

进入【邮箱管理】页面，点击"添加邮箱"：

  • 邮箱地址：完整邮箱地址，如 user@example.com
  • 密码：IMAP 登录密码
  • 所属域名：选择已添加的域名
  • 邮箱容量：磁盘空间限制（可选，0 表示不限制）

邮箱状态

状态说明
启用邮箱可正常收发邮件
禁用邮箱暂停使用，IMAP 登录和邮件投递均被阻止

邮箱操作

  • 修改密码：重置邮箱登录密码
  • 清理邮箱：删除该邮箱下的所有邮件数据
  • 启用/禁用：控制邮箱可用状态

过滤规则管理

过滤管线

EasyMail 的反垃圾过滤通过 Milter 协议与 Postfix 对接，分 6 个阶段处理每封入站邮件：

  1. Connect 阶段：记录连接 IP，进行 IP 信誉检查
  2. Helo 阶段：验证客户端 HELO/EHLO 主机名，检查反向 DNS
  3. MailFrom 阶段：验证发件人域名，检查 SPF 记录
  4. RcptTo 阶段：验证收件人是否存在，检查是否在联系人列表中
  5. Headers 阶段：分析邮件头部（Received、DKIM-Signature 等），移除入站 EasyMail 策略头
  6. Body 阶段：MIME 解析、文本/HTML 提取、URL 特征分析、附件风险评估、ML 分类推理、ClamAV 病毒扫描、规则引擎最终判定

过滤结果

结果动作说明
accept正常投递邮件进入收件箱
spam标记为垃圾邮件投递到垃圾邮件文件夹
quarantine隔离邮件暂存隔离区，需管理员人工审核
rejectSMTP 拒绝直接返回 5xx 错误码，邮件不入库

创建过滤规则

在【过滤规则】页面，点击"添加规则"：

  • 名称：规则的描述性名称
  • 条件：选择特征提取器和比较条件。例如：
    • 连接 IP 在已知恶意 IP 列表中
    • HELO 名称与反向 DNS 不匹配
    • 发件人域名无 MX 记录
    • 正文包含特定关键词
    • URL 链接指向已知恶意站点
    • 附件类型为高危格式
  • 动作：匹配后的处理方式（接受 / 标记垃圾 / 隔离 / 拒绝）
  • 优先级：数字越小优先级越高，规则按优先级顺序评估
  • 启用状态：可临时禁用某条规则

ML 分类模型

【分类模型】页面管理机器学习分类引擎：

  • 模型类型：FastText（基于词向量的文本分类）、DistilBERT（基于 Transformer 的深度分类器）、XGBoost（待实现）
  • 模型管理：导入、导出、训练、删除模型
  • 样本管理：在【训练样本】页面管理全局训练数据，用于模型训练
  • 训练：使用样本数据训练分类模型，训练完成后自动生效
  • 预测：对指定邮件内容进行预测，验证模型效果
提示 ML 分类是可选的。如果不配置模型，系统仅使用规则引擎进行过滤。

投递日志

【投递日志】页面记录每封邮件的处理详情：

  • 发件人、收件人
  • 邮件主题
  • 过滤时间
  • 过滤结果（accept / spam / quarantine / reject）
  • 匹配的规则
  • 特征提取快照
  • 投递状态

日志支持按时间范围、发件人、收件人、过滤结果进行筛选，方便排查问题。

管理员账户

在【个人资料】页面可以管理当前管理员账号：

  • 修改姓名、邮箱
  • 修改密码
  • 切换界面语言（支持中文/英文）
  • 切换界面主题

Postfix 管理页面

如果部署了 Postfix Agent，管理后台还提供以下 Postfix 管理功能：

  • 服务状态：查看 Postfix 运行状态和配置版本
  • 配置管理：远程管理 Postfix 配置参数，支持配置下发和回滚
  • 代理管理：管理 Postfix Agent 服务器
  • 队列管理：查看 Postfix 邮件队列

更多详情请参考 Postfix 集成。
