安装部署

环境要求

依赖版本说明
Postfix3.x边缘 MTA，处理 SMTP 入站。需要支持 Milter 和 LMTP
MySQL8.0+管理数据库，存储域名、账户、过滤规则等
Redis6.x+强烈推荐。用于缓存和过滤滑动窗口统计
Go1.25+仅编译时需要，运行时不依赖

获取 EasyMail

从源码编译

git clone https://github.com/your-org/easymail.git
cd easymail
go build -o easymail ./cmd/easymail

编译产物为单个二进制文件 easymail（或 easymail.exe），包含所有内置服务和协议适配器。

配置文件

EasyMail 使用 YAML 配置文件。创建 easymail.yaml：

# 基础配置
server_name: mail.example.com
storage_root: /var/easymail/data

# 数据库
database:
  driver: mysql
  dsn: "root:password@tcp(127.0.0.1:3306)/easymail?charset=utf8mb4&parseTime=True&loc=Local"
  automigrate: true

# Redis（可选但推荐）
redis:
  addr: "127.0.0.1:6379"
  password: ""
  db: 0

# 各服务端口
ports:
  admin: 8000      # 管理后台
  webmail: 8080    # Webmail
  imap: 143        # IMAP
  imaps: 993       # IMAP over TLS
  lmtp: 10027      # LMTP 投递
  milter: 10026    # Milter 过滤
  dovecot: 10025   # Dovecot 认证代理

# 管理后台静态文件路径
admin_static: ./frontend/admin/dist

# 日志
log_level: info    # debug / info / warn / error

# 反垃圾模块
filter:
  # ... 详见反垃圾过滤章节

heartbeat_second: 10  # 心跳间隔
提示 如果启用了 database.automigrate: true，首次启动时 EasyMail 会自动创建所有数据库表。

启动服务

./easymail -config easymail.yaml

启动后，EasyMail 会依次初始化数据库连接、创建表结构、加载缓存，然后启动所有配置的服务。日志输出示例：

验证服务状态

启动后，可以通过管理后台查看各服务状态：

打开浏览器访问 http://服务器IP:8000。首次登录使用默认管理员账号：

  • 初始账号：admin@easymail.local
  • 初始密码：在启动日志中查找 admin password 字段
安全提醒 首次登录后请立即修改管理员密码。生产环境建议配置 HTTPS（反向代理或直接配置 TLS 证书）。

登录后的控制面板会显示所有服务的运行状态：
