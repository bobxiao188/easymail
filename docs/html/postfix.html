Postfix 集成

EasyMail 不处理 SMTP 入站，依赖 Postfix 作为边缘 MTA。Postfix 通过 Milter 协议将邮件交给 EasyMail 做反垃圾过滤，过滤后的邮件通过 LMTP 投递到 EasyMail 存储。

集成架构

发送方 ──SMTP──→ Postfix (25/587)
                      │
                      ├── Milter (10026) ──→ EasyMail 过滤决策
                      │                        └── accept / reject / spam / quarantine
                      │
                      └── LMTP (10027) ──→ EasyMail 投递
                                             └── 写入 SQLite 索引 + 磁盘存储

安装 Postfix

apt install postfix
# 安装时选择 "Internet Site"，填写域名

配置 main.cf

编辑 /etc/postfix/main.cf，追加以下配置：

# 邮件域名查询（通过 EasyMail Dovecot 协议代理）
virtual_mailbox_domains = tcp:192.168.1.1:10025
virtual_mailbox_maps = tcp:192.168.1.1:10025

# LMTP 投递（将邮件交给 EasyMail 存储）
virtual_transport = lmtp:inet:192.168.1.1:10027

# Milter 反垃圾过滤
smtpd_milters = inet:192.168.1.1:10026
milter_default_action = accept

# SMTP 认证（允许客户端发信）
smtpd_sasl_type = dovecot
smtpd_sasl_path = inet:192.168.1.1:10025
smtpd_sasl_auth_enable = yes
smtpd_sasl_security_options = noanonymous

# 收件人限制
smtpd_recipient_restrictions =
    permit_mynetworks,
    permit_sasl_authenticated,
    reject_unauth_destination

# 不依赖本地用户数据库
local_recipient_maps =
重要 将 192.168.1.1 替换为 EasyMail 服务器的实际 IP 地址。EasyMail 的 Dovecot 协议代理同时提供三种服务：邮件域名查询、SASL 认证和 LMTP 投递。

配置详解

参数值作用
virtual_mailbox_domainstcp:IP:10025Postfix 通过 TCP 查询 EasyMail 确认域名是否有效
virtual_mailbox_mapstcp:IP:10025Postfix 查询邮箱地址是否存在于 EasyMail
virtual_transportlmtp:inet:IP:10027将邮件通过 LMTP 投递到 EasyMail
smtpd_miltersinet:IP:10026Milter 过滤链，EasyMail 分析邮件后返回决策
smtpd_sasl_*—SMTP 认证，客户端发信时需要登录邮箱账号
smtpd_recipient_restrictions—收件人限制规则，按顺序评估

master.cf

默认的 master.cf 即可工作，无需修改。

验证集成

1. 验证域名查询

postmap -q example.com tcp:192.168.1.1:10025

应返回 OK（如果域名已添加到 EasyMail 管理后台）。

2. 验证邮箱查询

postmap -q user@example.com tcp:192.168.1.1:10025

应返回 OK（如果邮箱已创建）。

3. 发送测试邮件

echo "Hello from test" | sendmail user@example.com

查看 EasyMail 日志确认邮件已投递。

4. 查看 Postfix 日志

tail -f /var/log/mail.log

常见 Postfix 配置

允许内部网络发信

mynetworks = 127.0.0.0/8 192.168.0.0/16 10.0.0.0/8

限制单封邮件大小

message_size_limit = 25600000  # 25MB

并发连接限制

smtpd_client_connection_count_limit = 10
smtpd_client_message_rate_limit = 30

TLS 配置（587 STARTTLS）

smtpd_tls_cert_file = /etc/ssl/certs/postfix.pem
smtpd_tls_key_file = /etc/ssl/private/postfix.key
smtpd_tls_security_level = may
smtpd_tls_loglevel = 1

Postfix Agent 远程管理

EasyMail 附带一个 postfix-agent 工具，部署在 Postfix 主机上，通过 HTTP 接口与 EasyMail 管理后台通信，实现对 Postfix 实例的远程管理。

工作原理

postfix-agent 在 Postfix 主机上启动一个 HTTP 服务，主动向 EasyMail 管理后台发起心跳注册。Agent 使用 --agent-token 进行身份验证，只有持有正确 token 的 agent 才能被管理后台识别和控制。

Postfix 主机                           EasyMail 服务器
┌──────────────────┐     HTTP/心跳     ┌────────────────────┐
│  postfix-agent   │ ────────────────→ │  管理后台 Web      │
│  --agent-token   │ ←──────────────── │  + API 服务        │
│                  │   配置/命令推送    │                    │
│  操作 postfix    │                   │  管理员 Web 操作    │
│  (postqueue,     │                   │                    │
│   postsuper,     │                   │                    │
│   postfix reload)│                   │                    │
└──────────────────┘                   └────────────────────┘

部署方式

在 Postfix 主机上下载 EasyMail 分发的 postfix-agent 二进制文件，运行：

./postfix-agent --agent-token=your-agent-token-here

参数说明：

参数说明
--agent-tokenAgent 身份令牌，在管理后台生成，用于认证
--easymail-addrEasyMail 服务器地址（默认 http://localhost:8080）
--listen本地监听地址（默认 :8081，用于健康检查）

建议使用 systemd 或 supervisord 管理 agent 进程，确保开机自启。启动后可在 agent 日志中看到类似输出：

[INFO] Registering with EasyMail at http://192.168.1.1:8080
[INFO] Agent registered successfully, id: agent-xxxx

Admin 后台 — 添加 Postfix Agent

在 EasyMail 管理后台中：

  1. 进入 系统设置 → Postfix Agents 页面
  2. 点击 添加 Agent
  3. 填写 Agent 名称（如 "主 Postfix 节点"）
  4. 系统生成一个 agent_token（如 ema_xxxxxxxxxxxx）
  5. 将此 token 复制到 Postfix 主机的 postfix-agent --agent-token=... 命令中
  6. 保存后，Agent 列表中会出现新条目，状态为 等待连接

Agent 启动并成功注册后，状态变为 在线，并显示最后心跳时间、IP 地址等信息。

Admin 后台 — Postfix Config 选择应用 EasyMail Host Address

当 Agent 注册成功后，在 Postfix 配置管理 中，选择一个已注册的 Agent，EasyMail 会将该 Agent 标记为目标部署主机：

  1. 进入 系统设置 → Postfix 配置
  2. 编辑或创建一条 Postfix 配置（main.cf / master.cf 片段）
  3. 在 应用目标 下拉框中选择一个已注册的 Agent（显示为 "Agent 名称 - IP 地址"）
  4. 系统会自动将 EasyMail 的 Host Address 注入到配置模板中，生成最终的 main.cf 配置

配置模板中可以使用 {{.EasyMailHost}} 变量引用 EasyMail 服务器的地址：

# 模板中的占位符会在推送时被替换为实际的 EasyMail 地址
virtual_mailbox_domains = tcp:{{.EasyMailHost}}:10025
virtual_mailbox_maps = tcp:{{.EasyMailHost}}:10025
virtual_transport = lmtp:inet:{{.EasyMailHost}}:10027
smtpd_milters = inet:{{.EasyMailHost}}:10026

推送并部署配置

配置编辑完成后：

  1. 在 Postfix 配置页面点击 部署
  2. 选择目标 Agent（勾选需要部署的 Postfix 主机）
  3. 确认配置变更预览，系统会显示 diff（新增/修改/删除的内容）
  4. 点击 确认推送

推送流程：

管理后台                    Agent                       Postfix
   │                         │                           │
   ├── 下发配置 ──────────→  │                           │
   │                         ├── 写入 /etc/postfix/      │
   │                         ├── 校验配置                │
   │                         │   postfix check           │
   │                         ├── 重载服务                │
   │                         │   postfix reload          │
   │                         ├── 返回结果 ──────────→    │
   │                         │                           │
   │  ── 显示部署结果 ──    │                           │
   │  成功/失败 + 日志       │                           │

每次推送都会记录部署历史，包括操作时间、操作人、配置变更摘要和执行结果。

队列管理

通过 Agent 可以在管理后台远程查看和管理 Postfix 邮件队列：

查看队列状态

  1. 进入 系统设置 → Postfix Agents
  2. 点击目标 Agent 的 队列管理 按钮
  3. 系统显示当前队列概览：
队列类型说明
active正在投递中的邮件
deferred投递失败等待重试的邮件
bounce退信队列
hold被搁置的邮件
incoming正在进入的邮件

队列操作

支持以下远程操作：

  • 刷新队列 — 立即重试所有 deferred 队列中的邮件（等价于 postqueue -f）
  • 删除邮件 — 按队列 ID 删除指定邮件（等价于 postsuper -d <queue_id>）
  • 搁置/释放 — 将邮件置入 hold 队列或从 hold 队列释放（等价于 postsuper -h/-H <queue_id>）
  • 清空队列 — 清空指定队列中的所有邮件（等价于 postsuper -d ALL）

所有队列操作均有操作审计记录。

实时日志

队列管理页面同时展示 Agent 的实时日志输出，方便排查投递问题：

2026/07/12 10:23:45 [mail.log] postfix/qmgr[12345]: AABBCCDD: from=<user@example.com>,
    size=12345, nrcpt=1 (queue active)
2026/07/12 10:23:46 [mail.log] postfix/smtp[12346]: AABBCCDD: to=<dest@other.com>,
    relay=mail.other.com[1.2.3.4]:25, delay=0.5, status=sent (250 2.0.0 Ok)
