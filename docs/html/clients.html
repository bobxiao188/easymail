邮件客户端配置

EasyMail 提供完整的 IMAP4rev1 服务，兼容主流邮件客户端。以下为各客户端的配置参数。

通用配置参数

参数值
IMAP 服务器你的 EasyMail 服务器 IP 或域名
IMAP 端口143（明文）或 993（SSL/TLS）
SMTP 服务器你的 Postfix 服务器 IP 或域名
SMTP 端口587（STARTTLS）或 25（明文）
用户名完整的邮箱地址，如 user@example.com
密码创建邮箱时设置的密码
认证方式普通密码（PLAIN / LOGIN）
注意 SMTP 端口 25 通常被 ISP 封锁，建议客户端使用 587 端口发送邮件。如果 IMAP 使用 993（SSL），SMTP 也建议使用 465 或 587 加密。

Foxmail 配置

1

打开 Foxmail，点击 "工具" → "账号管理" → "新建"

2

输入邮箱地址和密码，点击"手动设置"

3

填写服务器信息

字段值
接收服务器类型IMAP
接收服务器你的 EasyMail 服务器 IP
接收端口143
SSL不勾选（或勾选 993 端口）
发送服务器你的 Postfix 服务器 IP
发送端口587
发送服务器需要验证勾选
4

点击"测试"，确认连接成功后保存

Outlook 配置

1

打开 Outlook，点击 "文件" → "添加账户"

2

选择"手动设置或其他服务器类型"，点击"下一步"

3

选择"POP 或 IMAP"，点击"下一步"

4

填写账户信息

字段值
您的姓名显示名称
电子邮件地址user@example.com
账户类型IMAP
接收邮件服务器EasyMail 服务器 IP
发送邮件服务器Postfix 服务器 IP
用户名user@example.com（完整邮箱地址）
密码邮箱密码
5

点击"其他设置"，切换到"发送服务器"选项卡

  • 勾选"我的发送服务器(SMTP)要求验证"
  • 选择"使用与接收邮件服务器相同的设置"
6

点击"确定"返回，点击"下一步"测试配置

Thunderbird 配置

1

打开 Thunderbird，点击 "文件" → "新建" → "现有邮件账户"

2

输入名称、邮箱地址和密码，点击"继续"

3

Thunderbird 会自动尝试查找配置。如果失败，选择"手动配置"

字段值
收件协议IMAP
服务器主机名EasyMail 服务器 IP
端口143
SSL无（或 SSL/TLS 配合端口 993）
认证普通密码
发件服务器主机名Postfix 服务器 IP
端口587
SSLSTARTTLS
认证普通密码
4

点击"完成"，等待 Thunderbird 同步

手机客户端配置（iOS / Android）

iOS 邮件

  • 设置 → 邮件 → 账户 → 添加账户 → 其他
  • 添加 IMAP 账户，填写邮箱地址、密码
  • 收件服务器：EasyMail IP，端口 143，无 SSL
  • 发件服务器：Postfix IP，端口 587，STARTTLS
  • 用户名使用完整邮箱地址

Android（Gmail 客户端）

  • 打开 Gmail → 添加账户 → 其他
  • 输入邮箱地址
  • 选择 IMAP（不是个人 IMAP）
  • 输入密码
  • 收件服务器：EasyMail IP，端口 143，安全类型"无"
  • 发件服务器：Postfix IP，端口 587，安全类型"STARTTLS"
  • 用户名使用完整邮箱地址

验证配置

配置完成后：

  1. 尝试发送一封测试邮件给自己
  2. 稍等几秒后点击"接收"
  3. 检查收件箱是否收到邮件
  4. 回复测试邮件确认收发链路完整
常见问题 客户端配置不通过时，首先检查：
- EasyMail 和 Postfix 服务是否正常运行
- 邮箱账户是否已经创建并启用
- 防火墙是否放行了 143、587、25 端口
- 是否使用了完整的邮箱地址作为用户名
- 查看 EasyMail 日志和 Postfix 日志获取详细错误信息
