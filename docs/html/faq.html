常见问题

无法登录管理后台

可能的原因和解决方法：

  • 服务未启动：确认 EasyMail 进程正在运行，日志中无报错
  • 端口未开放：检查防火墙是否放行了 8000 端口
  • 默认管理员密码：在启动日志中查找 admin password 字段获取初始密码
  • 如果忘记密码，可以通过数据库直接重置管理员密码
客户端连接 IMAP 失败

请按以下顺序排查：

  • 确认 IMAP 服务是否在运行（检查控制面板服务状态或 EasyMail 日志）
  • 确认邮箱账户已创建且状态为"启用"
  • 确认使用的是完整的邮箱地址作为用户名（如 user@example.com，不是纯 user）
  • 确认防火墙放行了 143 端口（和 993 端口，如果使用 SSL）
  • 检查 EasyMail 日志中是否有 IMAP 连接失败的错误信息
  • 在服务器本地使用 telnet 测试：telnet 127.0.0.1 143，看是否能建立连接
注意 EasyMail 的 IMAP 支持 AUTHENTICATE PLAIN 和 LOGIN 认证方式。如果客户端配置为安全认证（如 CRAM-MD5），请改为普通密码认证。
邮件发送失败（SMTP 拒绝）

可能的原因：

  • Postfix 未正确配置或未运行
  • SMTP 认证不通过：确认用户名（完整邮箱地址）和密码正确
  • 发件人域名未在 EasyMail 中添加
  • Postfix 的 smtpd_recipient_restrictions 拒绝连接
  • 邮件被 Milter 过滤拒绝（检查投递日志中的过滤结果）
  • 端口被封：25 端口常被 ISP 封锁，建议使用 587 端口

排查方法：查看 Postfix 日志 /var/log/mail.log 和 EasyMail 日志中的 Milter 决策。

收不到外部发来的邮件

排查链路：

  • 检查 DNS MX 记录是否正确指向了 Postfix 服务器
  • 确认 Postfix 日志中外部发件人是否成功连接到服务器
  • 查看 EasyMail 投递日志确认邮件是否被过滤规则拦截（spam / quarantine / reject）
  • 检查 Spam 和 Quarantine 文件夹中是否有被误判的邮件
  • 检查存储磁盘空间是否充足
  • 确认邮箱容量未超限
Foxmail / Outlook 客户端连接后立即断开

这通常是 IMAP 协议兼容性问题。请检查：

  • EasyMail 版本是否为最新（兼容性问题可能已在后续版本修复）
  • 检查 EasyMail 日志中 IMAP 连接断开的具体错误信息
  • IMAP 响应格式必须严格遵循 RFC 3501 规范，部分客户端对格式敏感
  • 不要在 FETCH 响应中使用 BODY.PEEK 作为数据项名，应改为 BODY
  • 确认 RECENT 响应值不为 0（如果邮件已存在）
Milter 过滤导致正常邮件被误判

解决方案：

  • 在【过滤规则】页面调整规则的优先级或阈值
  • 将误判的邮件从垃圾箱移回收件箱，系统会学习
  • 将发件人加入白名单（通过自定义规则）
  • 调整 ML 分类模型的阈值
  • 收集误判样本用于重新训练分类模型
  • 临时禁用导致误判的规则，定位问题规则
如何备份和恢复邮件数据

EasyMail 的邮件数据包含三部分：

  • MySQL 数据库：域名、账户、过滤规则、分类模型等管理数据
    备份方法：mysqldump -u root -p easymail > easymail_backup.sql
  • 磁盘存储：原始 .eml 文件（位于配置的 storage_root 目录下）
    备份方法：直接复制目录，如 rsync -av /var/easymail/data/ /backup/
  • Redis 数据：缓存和统计（非必需，丢失后可自动重建）

恢复流程：先恢复 MySQL 数据，然后恢复磁盘文件，最后启动 EasyMail。Redis 数据会自动重建。

如何升级 EasyMail
# 拉取最新代码
git pull

# 重新编译
go build -o easymail ./cmd/easymail

# 备份当前二进制
cp /usr/local/bin/easymail /usr/local/bin/easymail.bak

# 替换二进制
cp easymail /usr/local/bin/easymail

# 重启服务
systemctl restart easymail
注意 升级前建议先备份 MySQL 和存储目录。database.automigrate: true 配置会在启动时自动应用数据库 schema 变更。
如何查看系统日志

EasyMail 默认输出日志到标准输出（stdout）。如果使用 systemd 管理：

journalctl -u easymail -f

日志级别在 easymail.yaml 中配置：

  • debug：详细调试信息（排查问题时使用）
  • info：常规运行信息（默认）
  • warn：仅警告和错误
  • error：仅错误

日志格式示例：

time="2026-07-08T10:30:00+08:00" level=info msg="imap: connection from 192.168.1.100:54321" module=imap
time="2026-07-08T10:30:01+08:00" level=info msg="lmtp: delivered mail to user@example.com" module=lmtp
time="2026-07-08T10:30:02+08:00" level=debug msg="[IMAP-DEBUG] C: C1 LOGIN user@example.com ***" module=imap
如何配置 HTTPS

EasyMail 的管理后台和 Webmail 默认使用明文 HTTP。生产环境建议配置 HTTPS：

方案一（推荐）：使用反向代理（Nginx / Caddy）

# Nginx 配置示例
server {
    listen 443 ssl;
    server_name admin.example.com;

    ssl_certificate /etc/ssl/certs/example.com.pem;
    ssl_certificate_key /etc/ssl/private/example.com.key;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

方案二：在代码层面支持 TLS（需自行实现证书加载配置）。

EasyMail 支持哪些数据库

通过 GORM ORM 支持：

  • MySQL 8.0+（默认，推荐）
  • PostgreSQL（修改配置文件中的驱动类型）
  • SQLite3（仅测试用，生产环境不推荐）

在配置文件中修改：

database:
  driver: mysql       # 可选: mysql / postgres / sqlite
  dsn: "..."
