EasyMail 企业邮件网关

开源、自托管的邮件安全网关。在 Postfix 前方提供反垃圾过滤、病毒扫描、ML 分类和管理后台，通过 LMTP 完成高效投递。

核心特性

管理后台

域名管理、邮箱账户管理、过滤规则、分类模型、投递日志，所有操作在一个 Web 界面完成。

反垃圾过滤

Milter 协议对接 Postfix，六阶段过滤管线：连接分析、HELO 检查、发件人验证、收件人匹配、头部分析、正文深度检测。

ML 分类引擎

内置 FastText / DistilBERT 分类器，支持 ONNX Runtime 推理，可自定义训练样本和模型。

病毒扫描

集成 ClamAV 防病毒引擎，对邮件附件进行实时扫描，支持连接池和熔断保护。

IMAP4rev1 服务

完整 IMAP 协议支持，兼容 Foxmail、Outlook、Thunderbird 等主流客户端。支持 IDLE 推送，邮件实时到达。

Webmail

浏览器端收发邮件，支持多标签管理、联系人、附件预览、邮件搜索。

服务监控

心跳机制自动检测各服务状态，控制面板可视化展示邮件流量、热门发件人、系统健康度。

快速上手

从部署到第一封邮件送达，全程约 30 分钟。清晰的业务流引导：域名 → 邮箱 → Postfix → 验证。

系统架构

EasyMail 不处理 SMTP 入站，依赖 Postfix 作为边缘 MTA。整体架构如下：

外部邮件 ──→ Postfix (SMTP)
                │
                ├── Milter (10026) ──→ 反垃圾过滤管线
                │                        ├── 连接 / HELO / 发件人 / 收件人检查
                │                        ├── 头部 / 正文深度分析
                │                        ├── ML 分类模型推理
                │                        └── ClamAV 病毒扫描
                │
                └── LMTP (10027) ──→ EasyMail 存储
                                        ├── MySQL（管理数据）
                                        ├── Redis（缓存 + 统计）
                                        ├── SQLite（每用户邮件索引）
                                        └── 磁盘（原始邮件 .eml 文件）
                                        │
                                        ├── IMAP (143/993)    ← 客户端收信
                                        ├── Webmail (8080)    ← 浏览器访问
                                        └── Dovecot (10025)   ← Postfix SASL 认证
