服务监控

EasyMail 使用心跳机制监控各服务组件的运行状态。每个启用的服务定期向 Redis 写入心跳时间戳，管理后台通过读取 Redis 中的心跳记录判断各服务的健康状态。

心跳机制

心跳机制的运作流程：

启动时
  │
  ├── 创建全局 ServiceManager
  ├── 注册各服务（Dovecot、LMTP、Milter、Admin、Webmail、IMAP、Classifier）
  │     └── 每个服务有独立的心跳间隔（默认继承全局配置）
  └── 启动心跳管理器
        │
        每 10 秒（可配置）：
        ├── 遍历所有注册的服务
        ├── 写入 Redis 当前时间戳
        │     └── key:  easymail:service:{服务名}:heartbeat
        │         value: ISO8601 时间戳
        └── 更新到下一轮

心跳配置在 easymail.yaml 中：

# 全局心跳配置
heartbeat_second: 10  # 心跳间隔（秒），设为 0 或未配置 Redis 时自动禁用

# 各服务配置（自动继承全局心跳设置）
imap:
  enable: true

milter:
  enable: true
说明 心跳自动启用（当 heartbeat_second > 0 且 Redis 可用时），无需在每个服务中单独配置。

管理控制面板

登录管理后台后，首页控制面板展示各服务的运行状态：

状态判断标准图标
running最后心跳时间 < 30 秒绿色
warning最后心跳时间 30-120 秒黄色
stopped最后心跳时间 > 120 秒 或 无心跳记录红色

控制面板还展示：

  • 今日邮件统计：总收件数、过滤数、拒绝数、垃圾邮件数
  • 热门发件人：今日发件量 TOP10
  • 服务状态：所有服务的一览状态

CLI 心跳工具

EasyMail 附带 heartbeat-cli 命令行工具，可快速查看所有服务的心跳状态：

cd cmd/heartbeat-cli
go run main.go -config /path/to/easymail.yaml

输出示例：

Service      Last Heartbeat          Age          Status
----------------------------------------------------------------------
dovecot      2026-07-08T10:30:45Z    3s           running
lmtp         2026-07-08T10:30:45Z    3s           running
milter       2026-07-08T10:30:45Z    3s           running
admin        2026-07-08T10:30:45Z    3s           running
webmail      2026-07-08T10:30:45Z    3s           running
imap         2026-07-08T10:30:45Z    3s           running
classifier   2026-07-08T10:30:45Z    3s           running

API 查询

管理后台的 GET /api/v1/dashboard/services 接口返回服务状态 JSON，可用于外部监控系统的集成：

GET /api/v1/dashboard/services
Authorization: Bearer {token}

响应：
{
  "code": 0,
  "data": {
    "dovecot": { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "lmtp":    { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "milter":  { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "admin":   { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "webmail": { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" },
    "imap":    { "status": "running", "lastHeartbeat": "2026-07-08T10:30:45Z" }
  }
}

Redis 直接查询

如果需要手动或通过脚本检查心跳状态：

# 列出所有注册的服务
redis-cli keys "easymail:service:*"

# 查看某个服务的心跳
redis-cli get "easymail:service:imap:heartbeat"

# 输出示例
# "2026-07-08T10:30:45Z"

邮件统计

管理后台的 API 还提供邮件流量统计：

  • GET /api/v1/dashboard/stats：返回今日邮件统计数据（总数、过滤数、拒绝数、垃圾数）
  • GET /api/v1/dashboard/top-senders：返回今日发件量 TOP10 的发件人

这些统计数据通过 Redis 计数器实时计算，不依赖数据库查询，响应速度快。
